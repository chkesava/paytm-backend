Welcome to the Paytm Backend API

This page provides detailed documentation on how to use the available API endpoints.

API Documentation

GET /api/counters

Fetches all counters with their current values.

Example Response: 
{
  "completed": 5,
  "halfCompleted": 3,
  "registeredNoPayment": 8,
  "wasted": 2
}

PUT /api/counters/:counter

Updates a specific counter by name.

Parameters:
  • counter (path parameter) - The name of the counter to update, e.g., completed, halfCompleted, etc.
  • value (body) - The new integer value for the counter.
Example Request (Updating 'completed' counter): 
PUT /api/counters/completed
{
  "value": 6
}

GET /api/entries

Retrieves all text entries.

Example Response: 
[
  { "_id": "6345345a33b8f3d1e0a17a5f", "text": "Sample entry 1" },
  { "_id": "6345345a33b8f3d1e0a17a60", "text": "Sample entry 2" }
]

POST /api/entries

Adds a new text entry.

Body Parameters:
  • text - A string containing the text for the new entry.
Example Request: 
POST /api/entries
{
  "text": "This is a new entry"
}

GET /api/status

Returns a summary of counter statuses with specific user names and their assigned statuses.

Example Response: 
[
  {
    "name": "User1",
    "status": "completed"
  },
  {
    "name": "User2",
    "status": "halfCompleted"
  }
]
